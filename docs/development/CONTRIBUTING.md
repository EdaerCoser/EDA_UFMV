## 贡献方式

### 报告问题

在[GitHub Issues](https://github.com/EdaerCoser/EDA_UFVM/issues)中报告bug或提出功能建议。

### 提交代码

1. Fork项目
2. 创建功能分支
3. 提交更改
4. 推送到分支
5. 创建Pull Request

### 改进文档

帮助完善文档也是宝贵的贡献！

---

## 开发规范

### 代码风格

- 遵循PEP 8规范
- 使用有意义的变量名
- 添加必要的注释和文档字符串

### 测试要求

- 为新功能添加单元测试
- 确保所有测试通过
- 测试覆盖率 > 80%

### 提交信息

使用清晰的提交信息格式：

```
<type>: <subject>

<body>
```

类型(type)：
- `feat`: 新功能
- `fix`: bug修复
- `docs`: 文档更新
- `test`: 测试相关
- `refactor`: 代码重构
