# 文档刷新流程指南

**版本**: v0.2.0
**更新日期**: 2026年3月1日

---

## 概述

本文档定义了当EDA_UFVM添加新功能后，文档更新的标准流程和顺序。遵循此流程可确保文档的一致性和完整性。

---

## 文档刷新顺序

### 完整刷新流程 (6步)

当完成新功能开发后，按以下顺序刷新文档：

```
1. 示例代码
   ↓
2. 测试文档
   ↓
3. API参考文档
   ↓
4. 使用指南
   ↓
5. 产品文档
   ↓
6. 项目主页 (README.md)
```

---

## 详细步骤

### 第1步: 创建/更新示例代码

**位置**: `examples/` 目录下对应功能子目录

**任务**:
- [ ] 创建可运行的示例代码
- [ ] 添加详细的注释说明
- [ ] 更新示例目录的README.md

**示例结构**:
```
examples/<feature>/
├── README.md          # 示例说明
├── basic_<feature>.py # 基础示例
├── advanced_<feature>.py # 高级示例
└── test_<feature>.py   # 测试用例
```

**检查点**:
- ✅ 示例代码可运行
- ✅ 输出结果清晰
- ✅ 注释充分

---

### 第2步: 更新测试文档

**位置**: `tests/test_<feature>/` 或测试文件头部

**任务**:
- [ ] 编写单元测试
- [ ] 编写集成测试
- [ ] 更新测试覆盖率说明

**示例**:
```python
"""
<功能> 单元测试

测试内容:
- <功能点1>
- <功能点2>
- 边界条件

作者: xxx
日期: 2026-03-01
"""
```

**检查点**:
- ✅ 测试通过率 > 90%
- ✅ 关键路径有测试覆盖

---

### 第3步: 编写API参考文档

**位置**: `docs/guides/<feature>/api-reference.md`

**任务**:
- [ ] 列出所有公开类和函数
- [ ] 记录参数说明
- [ ] 添加使用示例
- [ ] 说明返回值和异常

**模板**:
```markdown
# <功能> API参考

## 类1: ClassName

### __init__(self, param1, param2)

初始化<功能>对象。

**参数:**
- `param1` (type): 说明
- `param2` (type): 说明

**返回:** 对象实例

**异常:**
- `ValueError`: 当...

### method(self, arg1)

方法说明。

**示例:**
```python
obj = ClassName()
obj.method("value")
```
```

**检查点**:
- ✅ API文档完整
- ✅ 参数类型正确
- ✅ 示例代码可运行

---

### 第4步: 更新使用指南

**位置**: `docs/guides/<feature>.md` 或 `docs/guides/<feature>/README.md`

**任务**:
- [ ] 编写功能概述
- [ ] 说明核心概念
- [ ] 提供完整教程
- [ ] 添加最佳实践
- [ ] 包含故障排除

**章节模板**:
```markdown
# <功能> 使用指南

## 概述
<功能是什么，解决什么问题>

## 核心概念
<关键概念解释>

## 快速开始
<最小可运行示例>

## 详细教程
<分步骤说明>

## 最佳实践
<推荐做法>

## 故障排除
<常见问题和解决方案>

## 更多信息
<相关文档链接>
```

**检查点**:
- ✅ 新用户可理解
- ✅ 示例完整可用
- ✅ 涵盖常见场景

---

### 第5步: 更新产品文档

**位置**: `docs/product/` 目录下对应文档

**涉及文档**:
1. `docs/product/features.md` - 添加新功能说明
2. `docs/product/use-cases.md` - 添加应用场景
3. `docs/product/comparison.md` - 更新竞品对比（如需要）

**任务**:
- [ ] 在features.md中添加功能章节
- [ ] 在use-cases.md中添加典型用例
- [ ] 更新版本状态标记 (📋 → ✅)

**示例**:
```markdown
### N. <新功能> ✅ (v0.x.0)

#### 功能特性

- 特性1
- 特性2

#### 使用示例
```python
# 示例代码
```
```

**检查点**:
- ✅ 功能描述准确
- ✅ 版本号正确
- ✅ 状态标记更新

---

### 第6步: 更新项目主页

**位置**: `README.md` (根目录)

**任务**:
- [ ] 更新"主要特性"章节
- [ ] 添加示例链接
- [ ] 更新版本号徽章
- [ ] 更新路线图状态

**更新内容**:
```markdown
### 当前版本 (v0.x.0)

**<新功能模块>**:

- ✅ <功能1>
- ✅ <功能2>
```

**检查点**:
- ✅ 版本号一致
- ✅ 链接有效
- ✅ 简洁明了

## 不同类型功能的刷新重点

### 1. 核心功能模块 (如: v0.3.0 寄存器模型)

**刷新顺序**: 全部6步

**重点文档**:
- API参考 (详细)
- 使用指南 (完整教程)
- 产品文档 (功能说明)
- 示例代码 (基础+高级)

### 2. 增强功能 (如: v0.4.0 覆盖率引导)

**刷新顺序**: 1→2→3→4→6

**重点文档**:
- 使用指南 (集成到现有指南)
- 示例代码 (演示增强)
- 产品文档 (更新说明)

### 3. 小功能更新 (如: 新增bin类型)

**刷新顺序**: 1→3→4

**重点文档**:
- 示例代码 (简单演示)
- API参考 (参数说明)
- 使用指南 (功能章节)

### 4. Bug修复/性能优化

**刷新顺序**: 6→(可选) 4

**重点文档**:
- README.md (更新版本说明)
- CHANGELOG.md (记录修复)
- 使用指南 (如有行为变化)

## 文档刷新检查清单

### 开发完成时检查

- [ ] 示例代码可运行
- [ ] 测试全部通过
- [ ] API已实现并稳定
- [ ] 功能已验证

### 文档更新前检查

- [ ] 确定功能类型 (核心/增强/小功能)
- [ ] 确定刷新范围 (全部/部分)
- [ ] 准备示例代码
- [ ] 准备测试数据

### 文档更新后检查

- [ ] 所有示例代码可运行
- [ ] 所有链接有效
- [ ] 术语使用一致
- [ ] 版本号一致
- [ ] 状态标记正确 (✅/📋)
- [ ] 无明显拼写/语法错误

---

## 版本发布时文档检查

### Pre-release (发布前)

- [ ] 所有计划的文档已更新
- [ ] README.md版本号更新
- [ ] CHANGELOG.md更新
- [ ] 文档链接验证
- [ ] 示例代码测试

### Post-release (发布后)

- [ ] 用户反馈收集
- [ ] 文档改进记录
- [ ] 下次文档更新计划

**刷新范围**: 示例 → API → 使用指南

```
1. examples/coverage/advanced_coverage.py

## 文档维护最佳实践

1. **及时更新**: 功能完成的同时更新文档
2. **版本一致**: 确保版本号在各文档中一致
3. **链接验证**: 更新文档后检查所有链接
4. **示例测试**: 确保所有示例代码可运行
5. **术语统一**: 使用一致的术语和命名
6. **审查流程**: 重要文档更新后进行peer review
