## 1. 背景

### 1.1 现有问题

当前文档存在以下问题：
1. **不知道如何开始** - 新手缺乏清晰的入口
2. **SV概念映射不清楚** - SystemVerilog工程师不知道对应概念
3. **缺少完整示例** - 只有零散代码片段
4. **文档组织混乱** - 不知道先读哪篇
5. **技术深度不足** - 缺少实现框架和细节说明

### 1.2 设计目标

- ✅ 场景化组织 - 按"我想做什么"组织文档
- ✅ 渐进式技术深度 - 从快速开始到深入实现
- ✅ SV概念映射 - 帮助SystemVerilog工程师转换思维
- ✅ 完整可运行示例 - 每个场景都有可直接运行的代码
- ✅ 包含实现细节 - 框架、算法、设计模式、性能考虑

## 4. 场景清单

### 场景1：生成随机测试激励

**目标**：生成符合约束的随机激励测试DUT

**SV对应**：
- `rand/randc` 变量
- `constraint` 块
- `randomize()` 函数

**技术实现要点**：
- 双求解器架构（PurePython + Z3）
- 约束AST系统
- 装饰器模式API

**示例内容**：
- 简单随机变量
- 添加约束
- 控制分布（dist）
- 种子复现

---

### 场景2：收集功能覆盖率

**目标**：测量验证的完整性，找到未测试的边界情况

**SV对应**：
- `covergroup` / `coverpoint`
- `bins` 定义
- 交叉覆盖率 `cross`

**技术实现要点**：
- CoverGroup/CoverPoint/Cross层次结构
- 6种Bin类型实现
- 数据库后端（内存/文件）
- 报告生成器（HTML/JSON/UCIS）

**示例内容**：
- 定义覆盖率组
- 值bin和范围bin
- 交叉覆盖率
- 生成报告

---

### 场景3：创建寄存器模型

**目标**：为DUT创建Python寄存器模型，支持读写和字段访问

**SV对应**：
- `uvm_reg_field` / `uvm_reg` / `uvm_reg_block`
- UVM RGM方法（set/get/update/mirror）

**技术实现要点**：
- Field/Register/RegisterBlock层次化设计
- 15种访问类型（RW, RO, WO, W1C等）
- 硬件适配器（AXI/APB/UART/SSH）
- 代码生成器

**示例内容**：
- 定义寄存器块
- 添加字段
- FrontDoor/BackDoor访问
- 生成Verilog/C头文件

**示例内容**：
- 随机化迁移

## 5. 概念映射文档

### 5.1 SV↔Python映射表

核心参考文档，帮助SystemVerilog工程师快速找到对应概念：

| SystemVerilog | Python | 说明 |
|---------------|--------|------|
| `rand int x;` | `x: rand(int)(bits=32)` | 随机变量声明 |
| `constraint c { x > 0; }` | `@constraint def c(self): return self.x > 0` | 约束定义 |
| `covergroup cg;` | `cg = CoverGroup("cg")` | 覆盖率组 |
| `uvm_reg_block` | `RegisterBlock` | 寄存器块 |
| `std::randomize()` | `obj.randomize()` | 随机化调用 |

### 5.2 差异说明

深入解释关键差异：
- 静态类型 vs 动态类型
- 编译时 vs 运行时
- 仿真器语义 vs Python语义
