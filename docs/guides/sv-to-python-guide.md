# SV to Python 转换器用户指南

## 概述

SV to Python转换器将SystemVerilog中的UVM寄存器模型操作序列转换为使用EDA_UFMV RGM的Python配置脚本。

**版本**: v0.5.0
**状态**: Alpha

## 安装

```bash
pip install pyverilog jinja2 click
```

## 快速开始

### 基础用法

```bash
# 转换SV文件到Python
python -m sv_to_python convert dma_tasks.sv -o dma_tasks.py

# 查看文件中的tasks
python -m sv_to_python list dma_tasks.sv

# 验证生成的代码
python -m sv_to_python validate dma_tasks.py --check-syntax
```

### 命令选项

#### convert命令

```bash
python -m sv_to_python convert <input_file> [OPTIONS]
```

选项:
- `-o, --output PATH`: 输出文件路径
- `--task NAME`: 只转换指定的task
- `--verbose, -v`: 详细输出
- `--dry-run`: 只分析不生成文件
- `--reg-model NAME`: RGM模型名称（默认: reg_model）

#### list命令

```bash
python -m sv_to_python list <input_file> [OPTIONS]
```

选项:
- `--json`: 输出JSON格式
- `--detail`: 显示详细信息

#### validate命令

```bash
python -m sv_to_python validate <input_file> [OPTIONS]
```

选项:
- `--check-syntax`: 检查Python语法
- `--import-check`: 检查导入是否可用

## 支持的UVM模式

| UVM方法 | 转换结果 | 说明 |
|---------|----------|------|
| `write(status, val)` | `reg.write(val)` | 前门写 |
| `write(status, val, UVM_BACKDOOR)` | `reg.write(val, backdoor=True)` | 后门写 |
| `poke(status, val)` | `reg.write(val, backdoor=True)` | 后门写（别名） |
| `read(status, val)` | `val = reg.read()` | 前门读 |
| `peek(status, val)` | `val = reg.read(backdoor=True)` | 后门读 |
| `set(val)` | `reg.value = val` | 设置模型值 |
| `get()` | `val = reg.value` | 获取模型值 |
| `reset()` | `reg.reset()` | 复位寄存器 |

## 示例

### 输入SystemVerilog

```systemverilog
task automatic init_dma(
  input int channel,
  input logic [31:0] base_addr,
  input int length
);
  uvm_status_e status;

  // Enable DMA channel
  dma_reg_model.DMA_CTRL[channel].write(status, 32'h0000_0001, UVM_FRONTDOOR);

  // Set address and length
  dma_reg_model.DMA_ADDR[channel].write(status, base_addr);
  dma_reg_model.DMA_LEN[channel].write(status, length);

  // Start transfer
  dma_reg_model.DMA_CMD[channel].write(status, 32'h0000_0001);
endtask
```

## 高级用法

### 转换单个Task

```bash
python -m sv_to_python convert tasks.sv --task init_dma -o init_dma.py
```

### Dry-Run模式

```bash
python -m sv_to_python convert tasks.sv --dry-run
```

### JSON格式输出

```bash
python -m sv_to_python list tasks.sv --json
```

## 限制

- 复杂控制流（if/for/while）会生成注释，需要手动实现
- 动态值计算会保留变量名
- 自定义UVM扩展方法会生成TODO标记
- 不支持预processor指令

## 故障排除

### 转换失败

```bash
# 使用详细模式查看详细信息
python -m sv_to_python convert file.sv -o output.py --verbose
```

### TODO标记

生成的代码中的TODO标记需要手动实现：

```bash
# 检查TODO数量
python -m sv_to_python validate output.py
```

### 语法错误

如果生成的代码有语法错误：

```bash
# 验证语法
python -m sv_to_python validate output.py --check-syntax
```

## 集成到工作流

### Makefile集成

```makefile
# 从SV生成Python
%.py: %.sv
	python -m sv_to_python convert $< -o $@

# 批量转换
PYTHON_TASKS = $(patsubst %.sv,%.py,$(wildcard tasks/*.sv))

all: $(PYTHON_TASKS)
```

### 与RGM集成

生成的代码可以与EDA_UFMV的RGM系统配合使用：

```python
from rgm import RegisterBlock, Register, Field
from dma_tasks import init_dma

# 创建寄存器模型
dma_block = RegisterBlock("DMA", base_address=0x40000000)
# ... 添加寄存器定义

# 使用生成的函数
init_dma(channel=0, base_addr=0x1000_0000, length=1024)
```
